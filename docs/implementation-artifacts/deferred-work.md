# Deferred Work

## Deferred from: code review of 1-2-order-type-table-name-assignment (2026-06-21)

- **Seat release on SERVED/close (→ Story 1-8):** the occupancy FSM in 1-2 only drives `AVAILABLE↔OCCUPIED`. An order reaching `SERVED` drops out of `uq_table_active_order` (index says the table is free) but nothing sets `table_seat.status` back from `OCCUPIED` — so the status-based guard and the index can diverge, leaving a seat un-bookable via RPC. Resolve when close→DIRTY lands (Story 1-8): release/transition the seat in whatever RPC moves an order to `SERVED`/closed.
- **Table-swap deadlock window (concurrency hardening):** two concurrent `pos_assign_table` calls that swap the same two tables between two orders acquire the two `table_seat` row locks in opposite order → ABBA deadlock (Postgres aborts one; retryable, no corruption). The old seat is also released without a prior `SELECT … FOR UPDATE`. Fix by locking old+new seat rows in deterministic id order. Low probability at coffee-shop scale.
- **No-op reassign self-heal:** `pos_assign_table` early-returns when `v_order.table_id = p_table_id` without re-asserting the seat is `OCCUPIED`; a desynced seat would not be repaired. Largely moot once the release UPDATE is made idempotent (patch applied in 1-2), but a defensive re-assert could be added.
- **`public` in SECURITY DEFINER `search_path` (cross-cutting):** all POS RPCs use `set search_path = pos, catalog, app, public`. Including writable `public` in a definer function's path is a (minor) injection surface; builtins resolve via `pg_catalog` and all table refs are schema-qualified, so practical risk is low. Drop `public` (or qualify everything) across all RPCs in the RPC hardening pass.
- **No composite FK `(table_id, branch_id)` on `pos.orders`:** `orders.table_id` is a soft ref to `catalog.table_seat` with no FK; branch-consistency of an order's table is enforced only by RPC construction. A composite FK `(table_id, branch_id) → table_seat(id, branch_id)` would make cross-branch assignment structurally impossible. Schema hardening.
- **Test gaps (low):** AC6's partial-unique-index backstop is never exercised independently (the `status` guard always fires first); AC3's `CUSTOMER_NAME_REQUIRED` is covered by `01_orders_create.sql` but not re-asserted in `05_table_occupancy.sql`.

## Deferred from: code review of 1-1-order-creation-with-modifiers (2026-06-17)

- **Table integrity (✅ RESOLVED in Story 1-2, 2026-06-18):** `pos_create_order` now validates `p_table_id` against the caller's branch (`TABLE_NOT_FOUND`, also enforced by `catalog.table_seat` RLS), rejects a TAKEAWAY carrying a `table_id` (`TAKEAWAY_HAS_TABLE`), and raises the `TABLE_OCCUPIED` domain error (with `SELECT … FOR UPDATE`) before the raw `23505` index can fire. The global `uq_table_active_order` index is acceptable because `table_seat.id` is a global PK whose rows are now branch-scoped by RLS; cross-branch "squatting" is blocked by the in-RPC branch validation. See `20260618090001_table_occupancy.sql` and tests `05_table_occupancy.sql`.
- **Discount underflow (→ comp/discount story S-23):** if `discount_minor` ever exceeds the recomputed subtotal, `total_minor >= 0` CHECK throws `23514` and wedges the order with an opaque error. Discount is always 0 in story 1-1, so latent until discounts ship; add a `DISCOUNT_EXCEEDS_SUBTOTAL` guard then.
- **Idempotency (→ S-05):** `pos_create_order`/`pos_modify_lines` have no idempotency key; a retried request duplicates lines / re-creates orders. Covered when idempotency infrastructure lands.
- **Opaque error surface (cross-cutting hardening):** malformed input (non-UUID ids, non-array `p_ops`, bigint overflow on huge quantity, etc.) surfaces raw Postgres SQLSTATEs instead of named domain errors. Worth a consistent input-validation pass across all RPCs.
- **Auth defense-in-depth:** `app.is_admin()` and `created_by` trust JWT claims without re-validating against `app.user_account`; `current_claims()` is not fail-closed on malformed claims JSON. Acceptable under verified-JWT assumptions, but harden alongside the grants/RBAC story.
