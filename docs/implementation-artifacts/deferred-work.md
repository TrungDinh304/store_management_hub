## Deferred from: code review of 1-2-order-type-table-name-assignment (2026-06-21)

- **Seat release on SERVED/close (‚Üí Story 1-8):** the occupancy FSM in 1-2 only drives `AVAILABLE‚ÜîOCCUPIED`. An order reaching `SERVED` drops out of `uq_table_active_order` (index says the table is free) but nothing sets `table_seat.status` back from `OCCUPIED` ‚Äî so the status-based guard and the index can diverge, leaving a seat un-bookable via RPC. Resolve when close‚ÜíDIRTY lands (Story 1-8): release/transition the seat in whatever RPC moves an order to `SERVED`/closed.
- **Table-swap deadlock window (concurrency hardening):** two concurrent `pos_assign_table` calls that swap the same two tables between two orders acquire the two `table_seat` row locks in opposite order ‚Üí ABBA deadlock (Postgres aborts one; retryable, no corruption). The old seat is also released without a prior `SELECT ‚Ä¶ FOR UPDATE`. Fix by locking old+new seat rows in deterministic id order. Low probability at coffee-shop scale.
- **No-op reassign self-heal:** `pos_assign_table` early-returns when `v_order.table_id = p_table_id` without re-asserting the seat is `OCCUPIED`; a desynced seat would not be repaired. Largely moot once the release UPDATE is made idempotent (patch applied in 1-2), but a defensive re-assert could be added.
- **`public` in SECURITY DEFINER `search_path` (cross-cutting):** all POS RPCs use `set search_path = pos, catalog, app, public`. Including writable `public` in a definer function's path is a (minor) injection surface; builtins resolve via `pg_catalog` and all table refs are schema-qualified, so practical risk is low. Drop `public` (or qualify everything) across all RPCs in the RPC hardening pass.
- **No composite FK `(table_id, branch_id)` on `pos.orders`:** `orders.table_id` is a soft ref to `catalog.table_seat` with no FK; branch-consistency of an order's table is enforced only by RPC construction. A composite FK `(table_id, branch_id) ‚Üí table_seat(id, branch_id)` would make cross-branch assignment structurally impossible. Schema hardening.
- **Test gaps (low):** AC6's partial-unique-index backstop is never exercised independently (the `status` guard always fires first); AC3's `CUSTOMER_NAME_REQUIRED` is covered by `01_orders_create.sql` but not re-asserted in `05_table_occupancy.sql`.
